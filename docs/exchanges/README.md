## API Logging

All API requests and responses are logged locally using the `ExchangeApiLog` model in Prisma. Key features:

- **Full request/response logging** - All data is stored for analysis
- **Error tracking** - Error codes and messages are captured
- **Order rejection tracking** - Specifically flagged for analysis
- **Performance metrics** - Response times are tracked

### Usage

```typescript
import { logApiRequest, withApiLogging } from '@/lib/exchange-api-logger';

// Manual logging
await logApiRequest(
  {
    exchange: 'binance',
    endpoint: '/api/v3/order',
    method: 'POST',
    requestType: 'trade',
    params: { symbol: 'BTCUSDT', side: 'BUY' },
  },
  {
    status: 200,
    data: { orderId: '12345' },
    responseTime: 150,
  }
);

// Automatic logging wrapper
const result = await withApiLogging(
  {
    exchange: 'bybit',
    endpoint: '/v5/order/create',
    method: 'POST',
    category: 'linear',
    requestType: 'trade',
  },
  async () => {
    return await fetch('https://api.bybit.com/v5/order/create', { ... });
  }
);
```

## Best Practices

1. **Always use testnet first** - Test all logic on testnet before mainnet
2. **Implement proper error handling** - All API calls should have try/catch
3. **Rate limit management** - Respect exchange rate limits
4. **Time synchronization** - Use NTP for accurate timestamps
5. **Secure API keys** - Never expose keys in client-side code
6. **Log everything** - Use the logging service for debugging
7. **Use official SDKs when available** - They handle authentication and rate limits

## Regional Restrictions

- **Binance**: Some regions are restricted
- **Bybit**: IP addresses in US and Mainland China are blocked
- **OKX**: Some regions may have restrictions

Always check the exchange's terms of service for your region.
